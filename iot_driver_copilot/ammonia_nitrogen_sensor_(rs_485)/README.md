Ammonia Nitrogen Sensor (RS485) HTTP Driver

This driver connects to a VemSEE Ammonia Nitrogen Sensor (Modbus RTU over RS485) and exposes an HTTP interface to read the latest measurements. It continuously polls the sensor in the background, maintains a thread-safe buffer, and serves data via HTTP without exposing the native Modbus protocol.

Quick Start

1) Install dependencies:
   pip install -r requirements.txt

2) Set environment variables and run:
   HTTP_HOST=0.0.0.0 \
   HTTP_PORT=8000 \
   SERIAL_PORT=/dev/ttyUSB0 \
   MODBUS_BAUD=9600 \
   MODBUS_ADDRESS=1 \
   MODBUS_STOPBITS=1 \
   MODBUS_BYTESIZE=8 \
   MODBUS_PARITY=N \
   MODBUS_TIMEOUT_S=1.0 \
   SAMPLE_INTERVAL_MS=1000 \
   RETRY_MAX_ATTEMPTS=5 \
   RETRY_BASE_DELAY_MS=500 \
   RETRY_MAX_DELAY_MS=5000 \
   AMMONIA_REG=0 \
   AMMONIA_FUNC=input \
   AMMONIA_SCALE=0.01 \
   PH_REG=1 \
   PH_FUNC=input \
   PH_SCALE=0.01 \
   TEMP_REG=2 \
   TEMP_FUNC=input \
   TEMP_SCALE=0.1 \
   python3 driver.py

Notes:
- AMMONIA_OFFSET_REG/PH_OFFSET_REG and related FUNC/SCALE variables are optional; configure them if you need offsets returned.
- Register addresses (AMMONIA_REG, PH_REG, TEMP_REG) and function types (input/holding) must match your device's Modbus register map.

HTTP APIs

- GET /readings
  Description: Returns the latest measurements. Default fields: ammonia_ppm, ph, temperature_c.
  Optional query parameter: fields (comma-separated). Valid fields: ammonia_ppm, ph, temperature_c, ammonia_offset, ph_offset, device_address, baud_rate.
  Example:
    curl "http://localhost:8000/readings"
    curl "http://localhost:8000/readings?fields=ammonia_ppm,ph,device_address"

- GET /ph
  Description: Returns the latest pH value only.
  Example:
    curl "http://localhost:8000/ph"

Operational Behavior

- Background polling loop with timeout, retry, and exponential backoff.
- Thread-safe buffer stores the latest sample for fast HTTP responses.
- Logs connection, disconnection, errors, retries, and last update timestamps.
- Graceful shutdown stops background worker and closes Modbus client.
