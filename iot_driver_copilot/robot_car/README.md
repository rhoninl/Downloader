Robot Car HTTP Driver

This driver connects to a Tiangong SmartDonkey DR210 robot car using its native HTTP+JSON API, continuously polls telemetry in the background with retry/backoff, and exposes a minimal HTTP interface to command motion.

Only the explicitly required endpoints are implemented:
- POST /move/forward
- POST /move/backward

Each endpoint accepts a speed parameter and translates it to the device's native set_velocity command. Forward sends a positive linear velocity; backward sends a negative linear velocity. The driver maintains a resilient background telemetry polling loop but does not expose any additional endpoints.

Configuration (environment variables)
Set all of the following environment variables before running:
- HTTP_HOST: Host/IP for the driver HTTP server (e.g., 0.0.0.0)
- HTTP_PORT: Port for the driver HTTP server (e.g., 8080)
- DEVICE_BASE_URL: Base URL of the robot device (e.g., http://robot.local:8000)
- DEVICE_SET_VELOCITY_PATH: Native device path for velocity command (e.g., /api/v1/set_velocity)
- DEVICE_TELEMETRY_PATH: Native device path to poll telemetry (e.g., /api/v1/state)
- REQUEST_TIMEOUT_SECS: Request timeout applied to device HTTP operations (e.g., 5)
- POLL_INTERVAL_SECS: Telemetry polling interval when healthy (e.g., 0.5)
- BACKOFF_MIN_SECS: Minimum retry backoff on telemetry errors (e.g., 0.5)
- BACKOFF_MAX_SECS: Maximum retry backoff on telemetry errors (e.g., 10)
- MAX_SPEED_MPS: Absolute maximum linear speed allowed by the driver (m/s), used for validation (e.g., 1.2)

Optional authentication (choose one if required by your robot):
- DEVICE_BEARER_TOKEN: If set, the driver sends Authorization: Bearer <token>
- DEVICE_USERNAME and DEVICE_PASSWORD: If both set, the driver uses HTTP Basic Auth

Run
Install dependencies:
- Python 3.9+ is recommended
- pip install aiohttp

Start the driver (example):
HTTP_HOST=0.0.0.0 \
HTTP_PORT=8080 \
DEVICE_BASE_URL=http://robot.local:8000 \
DEVICE_SET_VELOCITY_PATH=/api/v1/set_velocity \
DEVICE_TELEMETRY_PATH=/api/v1/state \
REQUEST_TIMEOUT_SECS=5 \
POLL_INTERVAL_SECS=0.5 \
BACKOFF_MIN_SECS=0.5 \
BACKOFF_MAX_SECS=10 \
MAX_SPEED_MPS=1.2 \
python3 driver.py

HTTP API
- POST /move/forward
  - Parameters: speed (float, m/s), positive. Provide in JSON body or as query param.
  - Example (JSON body):
    curl -X POST http://localhost:8080/move/forward -H 'Content-Type: application/json' -d '{"speed": 0.5}'
  - Example (query):
    curl -X POST 'http://localhost:8080/move/forward?speed=0.5'

- POST /move/backward
  - Parameters: speed (float, m/s), positive. Provide in JSON body or as query param.
  - Example (JSON body):
    curl -X POST http://localhost:8080/move/backward -H 'Content-Type: application/json' -d '{"speed": 0.4}'
  - Example (query):
    curl -X POST 'http://localhost:8080/move/backward?speed=0.4'

Behavior
- The driver will POST the device's native set_velocity endpoint with JSON: {"linear_velocity": <value>, "angular_velocity": 0.0}.
- Telemetry is polled in the background from DEVICE_TELEMETRY_PATH with timeout, retry, and exponential backoff. Latest snapshot is buffered internally for responsive access by the driver (not exposed via API).
- Logs include connection/disconnection events, retries, errors, and last-update timestamps.
- Graceful shutdown handles SIGINT/SIGTERM and cleans up background tasks and HTTP sessions.

Notes
- This driver does not forward or redirect native URLs; it actively speaks the device protocol and exposes only the two required HTTP endpoints.
- Ensure DEVICE_SET_VELOCITY_PATH and DEVICE_TELEMETRY_PATH match your robot's HTTP API.
